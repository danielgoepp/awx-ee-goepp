# AWX Execution Environment - Goepp

This project contains the configuration and build files for creating a custom Ansible Execution Environment (EE) for AWX.

## Overview

Execution Environments are container images that provide a consistent, reproducible runtime for Ansible automation. This EE is customized for Goepp's specific automation requirements.

## Prerequisites

- Python 3.8 or higher
- Docker or Podman
- ansible-builder

## Setup

1. Create and activate the virtual environment:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # On Linux/Mac
   # .venv\Scripts\activate  # On Windows
   ```

2. Install ansible-builder:
   ```bash
   pip install ansible-builder
   ```

## Building the Execution Environment

To build the execution environment:

```bash
ansible-builder build -v3 --tag ghcr.io/danielgoepp/awx-ee-goepp:latest --container-runtime docker
```

## Current Configuration

- **Base Image**: CentOS Stream 9 (quay.io/centos/centos:stream9)
- **Python Version**: 3.11
- **Ansible Collections**:
  - awx.awx
  - ansible.posix
  - community.general
  - kubernetes.core
  - community.proxmox
- **Key Features**:
  - Helm 3 with helm-diff plugin
  - Git LFS support
  - Podman remote
  - Kerberos authentication support
  - Receptor integration
- **Python Dependencies**: Defined inline in execution-environment.yml (ncclient, paramiko, gssapi, pyOpenSSL, pywinrm, proxmoxer, etc.)
- **System Packages**: Git, Subversion, SSH clients, and compilation tools

## Usage

Once built, this execution environment can be:
- Pushed to a container registry
- Configured in AWX/Controller as an execution environment
- Used to run Ansible playbooks and automation
